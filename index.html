HTML5 Semantic Elements

The reasons why Semantic Tags Matters

Semantic tags brings about accessibility, best practices and structures, makes code easier to read and maintain,better understanding on your page structure through search engines (SEO) and better navigation through proper access.

Some of the HTML5 Semantic Elements include:"footer", "header", "nav","section" & "article".

Purpose of the Key HTML5 Semantic Tags

  • Main Tag - Represents the main content of the document.
  • Section Tag - offers a thematic grouping of meaningful content.
  • Article Tag - Represents a self-contained,reusable block.
  • Footer Tag - Marks the bottom section of a page.
  • Nav Tag - Contains the navigation link
  • Header Tag - Defines the top section of a page.

Web Accessibility Principles

Principles of Web Content Accessibility Guidelines (WCAG)

  • Understandable- the Content must be clear and easy to follow.
  • Operable- the users must be able to interact with all interfaces.
  • Perceivable- the information must be visible by the users.
  • Robust- it should be able to work reliably with the current and future assistive technologies.

HTML5 Accessibility Features

  • Semantic Elements like: header, nav, main, section, article, footer
  • ARIA landmarks and roles like: role:banner,header area
  • Form accessibility
  • Keyboard navigability
  • Proper heading structure (h1-h6)
  • Alt text for images
